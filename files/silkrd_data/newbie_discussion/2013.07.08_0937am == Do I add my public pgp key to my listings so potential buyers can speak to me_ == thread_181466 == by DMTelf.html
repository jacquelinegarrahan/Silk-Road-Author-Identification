Actually, anyone sending you an encrypted message uses your public key to encrypt, and you use your private key to decrypt.
The other way around, if you were to send a potential buyer a message, you use their public key to encrypt the message and they use their private key to decrypt the message again.
So both parties need eachother's public keys to be able to communicate securely.

public key: is public information and can be spread anywhere
private key: is private information and must be kept secret, preferably password protected and backups need to be kept on multiple devices.
