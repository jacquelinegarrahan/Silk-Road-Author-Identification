Title: Re: SR is up really fast right now
Post by: narcocapitalist9 on May 05, 2013, 08:04 am
http://dkn255hz262ypmii.onion/index.php?topic=155667.0

I just want to encourage everyone who can, to run a non-exit tor relay. But as I have discovered, raising the HTTP keep-alive timeout helps with the connection, because what it does is keep the connection to the server open instead of closing it and making a new one. This isn't important for clearnet sites so much, in fact it can be counterproductive, but for tor hidden services, which have much higher latency, and a higher cost of time and processing in establishing new connections (your tor client has to query the tor hidden service introducer node database every time it makes a new connection) it makes sense to reuse connections for hidden services rather than make new ones all the time. It does not have any significant disadvantage for the server itself, although it does lock up one connection per user that can't be recycled quickly, that is also a connection that can't be shuttered by a DoS attacker.
