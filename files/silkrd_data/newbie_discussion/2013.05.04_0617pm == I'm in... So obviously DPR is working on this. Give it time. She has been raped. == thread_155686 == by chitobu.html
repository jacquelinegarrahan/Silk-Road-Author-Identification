the more people who stump up some bandwidth to help increase tor capacity the more expensive the attack becomes, because the channel is wider and more legitimate users of sr will get through.

another thing that i have been thinknig might help is people raising the 'http keep-alive' time in their browsers. with a bigger number on that setting, a connection to SR will be locked away from the attacker, and available to you. the way 'keep-alive' works is the browser reuses connections to query the web server, if you set a longer time between making new connections, this also stops other users getting at the site. in this case, if you are a legitimate user of the site, then it might mean you can access the site for a lot longer.

it's not a long term solution, neither running tor non-exit relays nor the increasing of http keep-alive. but they will both raise the cost of the attack. and there is hundreds of us here. think about it. we could actually all help make sure that this attacker's yield is lowered. even if you personally can only lower their advantage by 0.1%, isn't that still worth it?
