I use GPG tools: https://gpgtools.org/

It's a bit confusing at first because it wants to hook up to your Mail app, but there are ways to get around this. Once it's installed, go to system preferences -> keyboard -> services and check off all of the OpenPGP keyboard shortcut options (eg OpenPGP encrypt selection = command+shift+E, etc). Now you're almost ready to encrypt a message. In the GPG app you downloaded you will have the option to import a public key. Take mine for example:

Well jeez, I feel like a jerk- I forgot my decryption password for the public key, so I can't decrypt your message. To answer your questions, signing is not necessary for us, especially as buyers. As for the latter, I'm not sure what you mean, so I can't quite give a satisfactory answer. I am a functional GPG user and should spend more time learning about its other features.

Regardless, this is how you decrypt a message:

Right click on your own line in GPG Keychain Access (it's bold and says: sec        [your name]          [your email]          [date created] etc..) -> export. Save it as whatever you like and open the file in TextEdit. That's your public key. Post it here and I'll use it to send you an encrypted message. You will decrypt it by pasting the message in TextEdit -> highlighting it beginning to end -> command+shift+D -> the passphrase you made the key with. Voila!

Post by: The Bubble Dreamer on May 01, 2013, 10:22 pm
On Mac, If you just want to encrypt, you can use simplegpgencrypt, a frontend for gpgtools, that makes it all very easy:

- copy public key (e.g. from vendor page) in the top first field
- type (or copy) your text (e.g. your address) in the second text field
- press the button
- copy the encrypted text from the third text field (and paste it e.g. in your order address on SR)

Easy does it, saves you the trouble of remembering command line stuff and is more user friendly than gpg keychain access.

It's open source and on google code. I checked the code and built it myself, but the provided binary looks good too.
