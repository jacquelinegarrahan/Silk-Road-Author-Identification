Post by: rogerjolly on October 03, 2013, 05:35 pm
At an absolute minimum, the new market will need to be designed with Kerchoff's principle in mind - even if entire server(s) and source code are seized or compromised, nothing major must be retrievable without knowing the private keys of the admin or individuals users.

In the NSA, there is a design philosophy that for every gadget X, the very first production copy gets sent straight to Kremlin.

I'm thinking that supplying a PGP key should be a mandatory piece of registration info for PMs (unless the user doesn't want access to PMs).  That way, if a user messages a vendor or another user, the message gets encrypted with their public key before being sent (preferably the user would do it themselves before it going out on the wire).  The recipient will get a bunch of ASCII gibberish that they can then decrypt to get the real message.  PITA for sure, but the fallout from SR seizure would've been FAR, FAR, FAR less if user communications weren't stored and sent in cleartext.

If you look at the affidavit, the feds would've had practically nothing on DPR except the fake IDs if the seized server image didn't spill all the beans.  That's why I'm not too keen on SM or BMR right now.  Redesigning security infrastructure like this takes weeks, if not months - and that's if you have developers well-versed in security.
