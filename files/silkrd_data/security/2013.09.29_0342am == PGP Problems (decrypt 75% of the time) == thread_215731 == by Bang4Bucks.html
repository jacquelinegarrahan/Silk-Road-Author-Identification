Abby is correct -- check the message for formatting -- ensure that it is cut/pasted properly, with all five dashes before and after the BEGIN PGP MESSAGE and END PGP MESSAGE lines.

Also, one thing to remember is that if the message has been produced by a mail client that contains HTML-formatted crap, this will often cause PGP to choke on it.

In cases where there is HTML (or sometimes high-ascii characters) you will need to decrypt it from the command-line. To do that, save the message to a file, go into a DOS box and issue the following commands:

gpg --decrypt saved_message_filename >decrypted_message

This should do the trick.

BTW, this will also show if the message is not encrypted to your key -- in that case, you'll get a different error message.
