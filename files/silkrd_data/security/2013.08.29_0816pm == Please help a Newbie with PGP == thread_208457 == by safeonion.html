I wouldn't use SR without it ;) all the data that transmits through here can be read in plain html.
download gnupg, gpg4win, or kleopatra. Tails liveusb comes with a great pgp client preinstalled in the OS.
get a public key, copy it, open a new text doc, paste, save as 'key' or something youll remember,
open your pgp client, and go to preferences -> import, then find the 'key' .txt and doubleclick.
the public key should now be in your list of keys. only use public keys you deem to be safe.
if you are in gpg4win/gnupg, go to the clipboard on the side, type a message and then go to options -> encrypt -> then pick that public key you imported, and double click. bam, one more layer of protection. if you are using tails, the gedit text editor will have the encrypt/decrypt options in it. if you are interested in making your own public keys i would highly suggest making the key higher than 2048 bits. 4096-bit is a strong key and won't be able to be broken for many many years. <1024 bit keys are on the verge of being able to be decrypted.
with the surge of attention focused on the bitcoin economy, the deepweb, freedomhosting, wikileaks, silkroad and more, this is really just should be standard procedure for safe communication. encryption end to end is your safest route, harddisk+software disk encryption(truecrypt/luks), keystroke encryption (keyscrambler) VPNs, tor with obfs3 bridges, on top of mixing those coins. im not even close to an expert but i just try to employ as many tools as i can.
