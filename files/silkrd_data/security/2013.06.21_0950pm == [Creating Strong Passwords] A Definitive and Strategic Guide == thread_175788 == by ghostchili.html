Title: [Creating Strong Passwords] A Definitive & Strategic Guide
Post by: ghostchili on June 21, 2013, 09:50 pm
I thought I'd share with you a set of rules to create a strong and different password for EVERY SINGLE site that you visit without having to write it down or remember it although they are all complex!  I'll explain with an example at the end.

Substitutions:
Code: [Select]
a = @
e = 3
i = 1 (one)
o = 0 (zero)
s = $

Create a pin code for yourself; an even number usually works best.  A pin number that you can remember!  For example, 1234 is a 4-digit pin number that we'll use (you can use a longer one if you'd like).

Now the general algorithm for the password is (and I'll explain):
Code: [Select]
1/2 Pin Number + Site Name (w/ Substitutions - first letter is capitalized and void of substitutions) + Special Charcter (I use ! ) + 1/2 Pin Number

With that we have our example pin number: 1234
Our exmple site is Facebook.com
Thus, the password would be: 12F@c3b00k!34

So it's a seemingly complicated password to others, harder to crack, and easy for you to remember!

Let's break down what happened.  We used the first two digits of our pin: 1234.  The site name is Facebook and if 'F' had a substitution you wouldn't use it here because you want at least one capital letter, one lower case letter, one number (our pin number), and special character - this encompasses them all!  So we use Facebook with our above substitutions.  At the end you append a special character (I like to use an exclamation point - personal preference).  Then we append the last half our pin: 1234.

So more examples of this would be:
12G00gl3!34 - for Google
12Tw1tt3r!34 - for Twitter
12P1nt3r3$t!34 - for Pinterest

See how I don't have to remember them.  But you DON'T have to use the site name.  The beauty with this is you can use anything and even write it down without worrying about a friend finding it and cracking it.  They don't know your rules (substitutions + pin splitting) and will never figure it out.

For example, I might write down "Account" as my password for Facebook.  But I still use my pin and substitutions for it.  This way you can leave it in plain site and never forget it!

Title: Re: [Creating Strong Passwords] A Definitive & Strategic Guide
Post by: kmfkewm on June 22, 2013, 12:03 pm
In practice though you can probably get away with slightly less. For one if the application you use the password with is using PBKDF with iterations, it will stretch the bit strength of the password out. A 19 character random ASCII password has 124.83 bits of entropy giving it a key space of 2^124.83 , if there is a PBKDF with 1,000 iterations the number of hash operations required to exhaust the key space will be 2^124.83 * 1,000 and log2(2^124.83 * 1,000) = 134.79 bit equivalency. An 18 character random ASCII password has a key space of 2^118.26 so with 1,000 PBKDF iterations it is log2(2^118.26 * 1,000) = 128.22 bit equivalency. So if you know the application you are entering your password into has 1,000 or more PBKDF iterations you are actually secure with only an 18 random character ASCII password.

Also, totally grammatically correct English prose is estimated to have about 1 bit per byte, but if you type things that you would never read in a book (ie: correct horse battery staple) and use special characters and such (ie: cOrrect h0rse batt3ry s!apl3) then the entropy significantly increases over 1 bit per byte, for the first few characters at least, although it levels off to near 1 bit per byte after many characters. So chances are you don't actually need 128 characters of human readable text, so long as it doesn't look like something you would write in a letter to somebody. Also the PBKDF will come into play for human readable passwords as well, so in the end you wont need quite 128 characters even if it does look like it is out of a letter or book.
