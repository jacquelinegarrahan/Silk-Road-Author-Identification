Make sure that there are FIVE dashes, i.e. ----- both before and after the BEGIN PGP MESSAGE. The same goes with the END PGP MESSAGE.
Even if just a single one of these dashes is missing, the message can fail to decrypt, or the key to import.
Also, if you're using email, look out for quoted-printable characters. PGP/GPG will sometimes complain about quoted-printable characters in a message. If you get such an error message, look for characters like:  =20 (space) and =3D (equals).
Strip out the =20 and 3D in front of the = sign, and you rmessage should decrypt normally.
