Title: Re: GPG Help please! - "Clipboard" contained no valid encrypted data
Post by: Party Girl on November 30, 2012, 12:35 am
Quote from: Guru on November 30, 2012, 12:28 am
Quote from: astor on November 30, 2012, 12:06 am
Doesn't GPG4USB generate weak keys by default? If you're using it for your own key generation, you need to pick larger bit keys (preferably 4096).

No. You're probably thinking of PortablePGP, which generates DSA/Elgamal 1024/512 by default.  GPG4USB generates RSA 2048/2048 by default, although you can generate larger ones if desired by clicking a button in a drop-down box. If you want to generate an RSA 4096/4096, it just takes 2-3 more mouse clicks.

Guru

Guru is definitely correct.  With those extra clicks, I am running RSA 4096.  ;)
