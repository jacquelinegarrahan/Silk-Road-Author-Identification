Title: Re: Is Liberte anonymous
Post by: Wadozo on January 28, 2013, 09:29 am
Quote from: d0z3r on January 28, 2013, 07:15 am
Is there a .onion site that does a test like that?

You do realize that by using Tor, your given a fake I.P address to browse the Tor network anonymously. .
Tor implements a form of onion routing to, basically, push encrypted data through multiple Tor nodes (servers), before it reaches the final destination (e.g. a website). The result is that neither the website owner, nor a local eavesdropper, nor any single Tor server knows who requested that specific website, hence you are communicating anonymously.
The biggest problem with many applications is that they leak DNS requests. That is, although they use Tor to anonymize the traffic, they first send a DNS request untorified in order to get the IP address of the target system. Then they communicate "anonymously" with that target. The problem: any eavesdropper with more than three brain cells can conclude what website you visited, if they see that you send a DNS request for rsf.org, followed by some "anonymous" Tor traffic.
If you use the Tor Browser Bundle (TBB), download the version appropriate to what operating system you're using. It comes pre-configured ready to browse the net anonymously using a fake I.P address, not the one given to you by your I.S.P.  :)

Title: Re: Is Liberte anonymous
Post by: northsouth on January 30, 2013, 04:21 am
You don't need the Tor Browser Bundle in Liberté Linux. To make the browser in Liberté safe, all you've gotta do is turn off JavaScript. All connections are forced through Privoxy, a local proxy which ensures all traffic is directed through Tor.

Also, there aren't any DNS leaks from Liberté anymore (at least not known), it was an issue in an older version, with a different browser called Midori (current browser is Epiphany). Besides, when visiting onion site, DNS leaks are not an issue, because DNS isn't used.

Quote from: d0z3r on January 30, 2013, 03:56 am
I was under the impression that using the tor bundle on your normal pc will leave traces of the sites you have been visiting. Is this true?

Only if you save bookmarks or passwords. Those will be written to files on the disk. Otherwise, it doesn't write to a cache on a disk during normal browsing. That would be a huge privacy leak. All browsing data is stored in RAM and discarded when TBB shuts down.
