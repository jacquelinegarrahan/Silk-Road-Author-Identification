Post by: CrunchyFrog on December 15, 2011, 03:15 am
Quote from: fosters
...Am I right in thinking that when encrypting, I should use the recipients public key somewhere?  If so, where/when/how?...
Yes.  Once you tell GPG to encrypt a message it will ask you to select the public key of the recipient from your keyring.  This means that his/her public key must already exist on your keyring at the time of encryption.

Assuming the recipient is an SR vendor, look at the vendor's profile to find their public key¹.  Copy (highlight, Ctrl-C) that text, paste (Ctrl-V) it into a *text* editor and save to disk.  Tell GPG to Import and it will ask you to indicate which file contains the key (the file you just saved to disk) to be added to your keyring.

Assuming all of that goes well, you should be ready to encrypt your message.

¹ Otherwise, search this thread [ dkn255hz262ypmii.onion/index.php?topic=174.0 ] or simply ask for it via PM.

Title: Re: Need help making sure I know how to use GPG
Post by: Aoth14 on December 16, 2011, 04:10 am
Get a modern PGP program. copy a members public key (whole thing from --- to ---) into a blank text file and save. open PGP program and find import key. click and find your txt file you copied the key into. The program will extract the key into your address book.Every time you write a message in the programs notepad/clipboard,etc, you hit the encrypt button, it will ask you who you are encrypting the message to,by bringing up the key address book. Choose the key of the person you're sending the message to, and hit ok. When you get the pgp message, copy and paste it to the person your sending it to. when they get it, they have a private key associated with the public key you encrypted the message with. they enter that key, and the message is decoded. thats how simple the process is.

 Now before you go sending messages,you need to create your own key. The public half of your key is what other users will use to code their message to you specifically. your private key is a pass-phrase you come up with, to decode messages sent to you (that were encrypted with your public code you gave them)

So once you have your own key. send it to whoever you want to send you encrypted messages.  Its good practice to paste your own public key in the bottom of your message BEFORE encrypting it, but you can also attatch your public key in the message after ----pgp msg end---  it just wont be encrypted
