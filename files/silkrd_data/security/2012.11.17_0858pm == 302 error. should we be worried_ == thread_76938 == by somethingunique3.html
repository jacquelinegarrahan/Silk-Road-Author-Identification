It's probably a redirect to the login page from the main site URL.

This guy gets it. silkroadvb5piz3r.onion is redirecting to silkroadvb5piz3r.onion/silkroad/home

That's why you're getting a 302 error.

I understand why an "error" might spook you, but think of it this way. I'm sure you've seen 404 errors. That means the web page doesn't exist. If you get a 404 error, does that mean the site has been compromised? Of course not. Web servers have dozens of error codes that they send, and 302 is one of them. In the case of 302, it means that the resource you request (silkroadvb5piz3r.onion) is being redirected to another resource (silkroadvb5piz3r.onion/silkroad/home). Nothing malicious about that. Happens all the time.
