Title: Increase anonymity by removing PGP metadata
Post by: acidrock on May 29, 2012, 06:11 am
I don't know if this has been mentioned before, but I see a lot of people posting their public keys with metadata like the "Version" and "Comment" lines. This reduces our anonymity, especially for people who use obscure programs. I suggest we make a habit of removing the metadata, which is easy to do. For gpg and related programs, the flag is --no-emit-version.  Like this

gpg --export --armor --no-emit-version acidrock@tormail.org

Notice how the version / comment lines are missing? You can add that to your public keys and it works just fine.

This flag also works for encryption. You can add it to Enigmail's "Additional parameters for OpenPGP" so it can transparently encrypt messages without the metadata (Enigmail has a nasty habit of advertising itself in the Comment line, which can also be disabled).

Alternatively, you can remove the metadata lines manually.
