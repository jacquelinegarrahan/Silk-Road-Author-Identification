Title: Re: Dec 13 14:27:35.553 [Warning] Query '[scrubbed]' didn't have valid rend desc in
Post by: astor on December 13, 2012, 08:43 pm
"rend desc" is the rendezvous descriptor, which tells you the hidden service introduction points.

For more on how hidden services work: https://www.torproject.org/docs/hidden-services.html.en

If you don't have a valid rendezvous descriptor for the HS, then your client doesn't know where to meet it.

You might try clearing all cached descriptors and forcing it to download again.
Title: Re: Dec 13 14:27:35.553 [Warning] Query '[scrubbed]' didn't have valid rend desc in
Post by: astor on December 13, 2012, 08:49 pm
A lot of people are reporting that SR is down

http://dkn255hz262ypmii.onion/index.php?topic=93352.0

Based on information in this email:

http://www.mail-archive.com/tor-bugs@lists.torproject.org/msg00010.html

 "When a hidden service is restarted the old intro points are not valid
 anymore. Tor seems to notice this and will fetch a new descriptor when it
 has run out of intro points but then after doing this expensive operation
 it simply discards it with the excuse that there is already *some*
 descriptor there. The expensive operation of fetching has been done for
 nothing.

 The next time it will again try the old descriptor of which it already
 *knows* that there are no valid intro points and fetch a new one (possibly
 better one) again, but only to discard it without even looking at it,
 again and again and again."


So I think you're getting that message because SR is either down or got restarted and your client can't connect to it at the intro points.
