Title: GPG Kleopatra VS WinPT
Post by: dss3i on September 30, 2012, 03:19 am
I decided to learn how to use GPG and downloaded GPG4win. I had planned to follow the guide by OMVD (something like that), but it turned out that already at the first step I couldn't check WinPT as he suggested, because it wasn't there. Instead I followed the compendium that came along with the GPG4win. It took me A LOT of time to manage to encrypt, decrypt, sign and verify messages, but I finally learned doing it with help of Kleopatra and Claw Mail, as recommended by the compendium. I came to think of a few obstacles while learning, such as having to decrypt messages if I want to store them, since Claw Mail won't encrypt the address (or subject, but hopefully nobody would put a revealing subject), and so if someone looked at my computer it would be possible to see that contact had been established between me and the other part, even though what we said wouldn't be able to be seen. Encrypting stored mails seemed quite tedious to me, so I came here to find solutions, and I realized that if using WinPT instead of Kleopatra, you won't need a mail client at all, because you can paste encrypted messages using the message system on SR, and WinTP will do the work without being connected to a mail client. This solves the problem of having to encrypt stored messages on the computer, since there will be no such thing. Also, there's no need for an anonymous e-mail to send the messages from, and less risk to confuse an anonymous certificate for SR, with a non-anonymous certificate for trading bitcoins. WinPT (or equivalent) seems a lot better for SR purposes in other words...so, I'm asking two things:

1. From what I've written, is there anything you can tell that I've misunderstod about GPG (I am trying to understand how it works so please explain my thought errors)?
2. How can I simplest learn how to use GPG in SR fashion, be it with WinPT or some other program (or is it perhaps possible with Kleopatra but I'm too stupid to figure it out)? :)
