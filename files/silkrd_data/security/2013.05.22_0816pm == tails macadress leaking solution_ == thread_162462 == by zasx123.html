Title: tails macadress leaking solution?
Post by: zasx123 on May 22, 2013, 08:16 pm
There was a thread about this but it is gone? The thing is:
Your mac adress DOES leak before even connecting to a network, so if tails starts up it immediately leaks to all wifi stations close to you. I am going to use tails in a public wifi so i dont want it to leak my own mac adress there. I found there is a command to shut wifi down and up in linux so a solution could be:
start tails at home, leaking is not a problem here because you use your original mac adress here always.
Then you do:  ifconfig wlan0 down
wifi is now shut down so no leaking or connections possible
you change the macadress at home/ or at the public wifi it does not matter
ifconfig wlan0 up so the wifi is running again.

This way your original mac adress can not leak right (except for near your house but this is not a problem)
