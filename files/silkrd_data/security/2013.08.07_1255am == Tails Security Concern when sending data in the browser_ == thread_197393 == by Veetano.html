Title: Re: Tails Security Concern when sending data in the browser?
Post by: GrimWaldo on August 07, 2013, 02:03 am
*sigh*
Tor is designed to triple-layer-encrypt every packet of data before sending it. It then chooses a psuedo-random 3-tier path for the data packet through the network. Each point of transmission peels off a successive layer of encryption until only the last server (the 'exit node') sends your actual unencrypted data. Kinda like an onion.

SR is what's known as a 'hidden service'. It lies inside of the Tor network... before any of the 'exit nodes'. ALL data comes to and from SR encrypted. What IceWeasle is actually telling you is that you're attempting to connect to a non-HTTPS website... which is true... but irrelevant... because all 'hidden services' are inherently encrypted. Same goes for the "you are about to send unencrypted info" message that pops up, just uncheck the box and hit OK. Tor is ALWAYS encrypted, except when leaving the 'exit node'.

I think they've decided to de-activate the warnings by default on the next version of TAILS. I sure hope so.
