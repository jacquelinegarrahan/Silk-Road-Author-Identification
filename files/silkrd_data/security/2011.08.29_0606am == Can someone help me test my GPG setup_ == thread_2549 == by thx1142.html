Email clients can be configured to encrypt and decrypt email messages on the fly, much like chat or other applications that are used to move text or data around.  In and of itself GPG is only responsible for the encryption of the data itself, it is not a communications protocol.  The email address field of GPG is just informational text.  It is not necessary and can be omitted, as you will notice mine has no email address.

Make sure you are including the lines with "-----BEGIN PGP MESSAGE-----" all the way to the end "-----END PGP MESSAGE-----" and everything in between.  Put all of that into a text file and it should let you decrypt it from the command line (depending on how it is set up).  Try naming the file with the extension .asc instead of txt.  This may be necessary to get it to recognise the file format.  I don't know for sure as I have never used that tool.  I just know the command-line commands.

Try this:

gpg -d -o - FILENAME.asc

That command should decrypt the file and display it to the screen (that's what -o - does, make sure to have the dash after the -o since a single minus sign means display on the screen instead of creating a file). 

You should be able to encrypt files with the following line:

gpg -ea -o - FILENAME.txt

If it is the same as the linux command line then you should be asked a few questions, first your password, then the recipients, just use the name from their key when you imported it. 

I'd look around and see if there is a better GUI tool for GPG on a mac.  They all use the same GPG backend so it should not be a big deal switching between them as the key stores should show up in whatever GUI application you use.
