Hi Anonymous,

The answers to your questions are quite big so I'll give the abbreviated direct response first to see what questions you'll ask next  ;)

You can encrypt messages using an application called PGP (Pretty Good Protection).
You should use encryption when giving a vendor your mailing address to receive your order.
You don't need to use encryption in daily postings such as on this forum or in messages with your vendor, but you should always treat any non-encrypted messages to be viewable by the public and therefore never say anything identifying about yourself.

Anonymous5000,

There a lot of tutorials on how to install and use PGP in this forum, but you're on the right path.When running the install you don't have to include Kleopatra, but do make sure you install the GPA components.
Once it's installed and the links are on your desktop go ahead and create a public key. Another forum member named Pine described the Public/Private keys much like an email address and password. Think of the Public key as your personal email address; you can send it to anyone, anyone can see it and therefore write to it BUT only you will be able to decrypt what they send to you using your Private key (password). 

To sum it up very simply; to send me an encrypted message you will need my Public Key, once you know what it is you use PGP to generate an encrypted message that only my private key can decode. For me to send you an encrypted message I need your Public key so that my PGP software can do the same.

Speaking of Pine, she has a PGP club for aspiring encryptors, join this thread when you think you have a grip on how it all works.
