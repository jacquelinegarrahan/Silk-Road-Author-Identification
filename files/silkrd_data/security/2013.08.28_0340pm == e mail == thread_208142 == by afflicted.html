Title: Re: e mail
Post by: dotgoat on August 29, 2013, 12:36 am
I assume this laptop has full disk encryption?  If not I'd do that.  If it's using tails I'm assuming that would set it up.  If it's windows then I'd recommend truecrypt, nice thing about truecrypt is it will do the encryption on the fly (after a couple minutes to do some initial encryption) so you don't have to reconfigure things.

For email I think it's pretty much use PMs here or on SR.  There's bitmessage but keep in mind like onion.to there's a bitmessage gateway at bitmessage.ch.  So people from clearnet could send a message to you.  But it's the same as onion.to.  The person using the gateway is the one that's getting easily identified, you're hiding behind a bitmessage address.  I tested sending to a non bitmessage.ch bitmessage address from a bitmessage.ch account and it worked.  I have not tested sending from a gmail account or something to <an address created on the windows client and not here>@bitmessage.ch to see if it forwards it on to the bitmessage p2p network.
