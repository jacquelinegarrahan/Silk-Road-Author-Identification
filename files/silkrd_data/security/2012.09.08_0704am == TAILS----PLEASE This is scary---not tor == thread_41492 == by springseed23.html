Title: Re: TAILS----PLEASE This is scary---not tor
Post by: raven92 on September 08, 2012, 07:32 am
Tor handles encryption/decryption of .onion sites in a different manner than SSL. Between Tor->firefox the data is not encrypted, and firefox realize that it is.

This has nothing to do with TAILS, its just how onion sites work on Tor.
Title: Re: TAILS----PLEASE This is scary---not tor
Post by: anonaddict on September 08, 2012, 01:43 pm
Correct. Actually having a .onion site use SSL would make it less secure and vulnerable to detection or location within the tor network. Si if it's a .onion site it is better than SSL.

Check and make sure you have HTTPS Everywhere installed as a firefox add on. If you are using tor browser bundle that it's there already.
