Title: Re: Introduction and a n00b question about PGP
Post by: sbaxter on January 11, 2012, 03:32 pm
Hi,

no it is absolutely not necessary to run gpg (the opensource variety of pgp) on the same computer as the browser you access SR with. You can do that, it might not even be a bad idea to keep you keyring on the USB-Stick, but it is not strictly necessary.

SR/Tor/GPG are not connected. Tor is a systrem of peer-to-peer exchanges that know only their immediate peers and encrypt traffice between them, to give you some anonymity. SR is a "normal" webserver that is only reachable (well, sort of) via Tor, and GPG/PGP is a method to encrypt messages. You can use all of them independently.

Yes, you need a program to use gpg, it is called "gpg" ;) Try to search for it in your packet-manager (synaptics, if you are using Ubuntu).
If you have that and still have questions, feel free to ask me, I am using gpg on the commandline, too.

You will probably want some anonymous email service, too, loike hushmail.com or tormail.com (and there are a lot more).

Using GPG is a REALLY GOOD idea! Not only that an in-between party (like the mailserver, SR, etc.) can read your message (e.g. address details), but you can also be sure that only the intended receiver can decrypt the message. So if you inadvertently send some secret data to the wrong recipient there is no harm done, as he cannot decrypt them anyway!
