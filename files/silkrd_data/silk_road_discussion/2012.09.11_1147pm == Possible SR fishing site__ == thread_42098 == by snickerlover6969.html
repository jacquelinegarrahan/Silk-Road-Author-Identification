Hi snickerlover6969,

No, that isn't a fishing site, but you're right to exercise caution!
The Startpage search result is simply an onion.to page. You can read more about onion.to on their website www.onion.to
Read the Disclaimer, specifically:

Quote from: onion.to
By using this service instead of connecting directly to the Tor network you are trading off security and anonymity for convenience. Know that we can identify the IP of visitors using this service (but chooses not to). We strongly recommended to access Tor Hidden Services directly using the Tor Browser Bundle to increase your security and anonymity.

Accessing it through Tor is no big deal, but there's no reason to do so. The reason that it showed the log in page when using onion.to is because you were attempting to reach the site through a different Tor node than the node your current Tor connection was routing through.
When you are having connection errors, hit "Use a New Identity" in the Vidalia Control Panel. You may have to do this a number of times before you see the page, or you may only have to do it once or twice.

You can use the following site to see if Silk Road is up or down: http://zw3crggtadila2sg.onion/downornot/index.php?url=silkroadvb5piz3r.onion
"Yes (but with 302 error)" means that the site is currently up.

You're very welcome.  :)

If you were using Startpage on the clearnet (i.e. not through Tor) or you accessed onion.to through a browser other than TorBrowser (Firefox, Safari, Chrome, Opera, IE, etc.), then yes, Startpage and onion.to will know your real IP address.

However, if you're using the Tor Browser Bundle (Vidalia) and searched using Startpage in the upper right hand corner, then no, Startpage doesn't know your real IP address. Similarly, if you access onion.to with TorBrowser then they won't know your real IP address either. It just means you're being routed through two different Tor nodes, the one your TorBrowser is currently using and the one the onion.to site runs you through.
