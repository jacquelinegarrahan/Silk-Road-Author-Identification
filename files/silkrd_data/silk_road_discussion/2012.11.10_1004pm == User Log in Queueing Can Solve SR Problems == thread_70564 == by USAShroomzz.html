Title: User Log in Queueing Can Solve SR Problems
Post by: USAShroomzz on November 10, 2012, 10:04 pm
I know how to solve the overloading problem SR is having.  The SR log in page needs to have a server running a queue system.  If you have to wait for in line you will automatically remove many people just trying to bust in the door F5ing over and over.  A separate server running the queue system would not even have to be anonymous.  It can be made where if a person never logs in within like a 1 minute time span, even though their queue number is 0 or a upper limit it will automatically boot them to last in line, thus preventing hanging user in queue line.  The limit can be dynamically adjusted to allow more people in at once depending on volume of traffic.
