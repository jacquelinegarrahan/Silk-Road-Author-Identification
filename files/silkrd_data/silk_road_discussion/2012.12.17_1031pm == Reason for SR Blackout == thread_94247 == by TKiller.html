For God's sake, people... look, this is not a normal internet site with a server that you get efficiently routed to like google.com or something.  This is a hidden Tor service.  That means you go through several computers on the way there, each of which have their own standard routing procedure -- fuck, how to put this...

It's like connecting to Google.com, then using THAT connection to connect to Yahoo.com, then using THAT connection to connect to askjeeves.com, then using THAT connection to connect to aol.com, and then linking through all of those encrypted connections to interact with aol.com.  If any of them are down, then the site will appear to be down.

Other people can get to SR (which is aol.com in my example)?  Then the site ain't down.  Your Tor program is just having trouble finding a route to it.  The SR server could be a billion dollar behemoth that could calculate more numbers in a nanosecond than Data from Star Trek -- if one of the links inbetween sucks, the site will appear to be fucking slow as molasses.

Make sense?  As for the "bad login," for some reason, I get it precisely once every single time.  No matter what.  And only once.  That's all I can say about it.
