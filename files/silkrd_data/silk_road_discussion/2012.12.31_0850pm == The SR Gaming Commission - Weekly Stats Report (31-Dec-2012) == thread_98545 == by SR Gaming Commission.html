Sure.

In this context (that is, md5 hash values), "digit" refers to a HEX, or base-16 digit. That includes 0-9 and a-f.
There is no algorithm other than a simple summing used to obtain these frequency charts.

We take a list of all transaction values, in this case 289 transaction ids and sum the appearance of each hex digit 0-f.

If you'd like to put the transaction ids we listed in our last report into Excel, the following formula would determine hex digit frequency:
=SUMPRODUCT((SUBTOTAL(3,OFFSET($A$1,ROW($A$1:$A$1000)-MIN(ROW($A$1:$A$1000)),,1))),--(LEN($A$1:$A$1000)-LEN(SUBSTITUTE($A$1:$A$1000,"HEX_DIGIT",""))))

where HEX_DIGIT equals the hex digit for which you wish to count frequency.
