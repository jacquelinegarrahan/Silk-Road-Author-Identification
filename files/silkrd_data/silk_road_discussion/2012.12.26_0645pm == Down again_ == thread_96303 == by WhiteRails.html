1) Well I don't know what OS you're running, but all of the ones of which I am aware have some kind of clock application, no? And therein will be some option to sync to an Internet clock. For example: I'm on Windows 7. There's a wee digital clock in my tray. I click on it, which brings up a calendar and the option to "Change date and time settings..." This brings up the "Date and Time" application window, which has a tab for "Internet Time." It will say that "This computer is set to automatically synchronize with 'subdomain.domain.TLD'. However, sometimes these syncs are erroneous due to timeout errors or whatever, so if I'm having trouble connecting to SR or if my laptop clock doesn't match my cell phone I go to "Change settings," choose whatever reference, and hit "Update Now." Sometimes the sync will timeout, so I move on to a different reference (some five are listed by default). I know on Linux (depending on distro) will be even more straightforward in this regard, if a GUI is being used at least, and will allow the option for adding custom references.

2) I actually don't know why this works. But someone on the forums said it once and it seems to really work for me if connection problems are intractable. There's no point in doing it more than once per say 12 hours though, unlike how one would use "Change Identity." Anyway I've been connecting much more successfully and consistently since I started using this trick. I suspect it has something to do with using the system clock to calculate timeouts or sthg I dunno.

Post by: chipmunk on December 29, 2012, 10:26 pm
Quote from: Lubriderm on December 28, 2012, 03:53 pm
Second, how would that help you get onto SR?

All modern computer security is based around encrypted tokens that expire, that way even if someone trying to intercept your communication were to crack a token you sent, and try to use it to pretend they were you, it would do them no good because no computer would accept it anymore.

If your computer time is inaccurate, all the other computers on the network will ignore your data because as far as they can tell, it has already expired, or it is invalid (from the future).
